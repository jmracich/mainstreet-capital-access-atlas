Accessibility

Accessibility is part of the project quality bar.

Main Street Capital Access Atlas is intended for public-interest use by community partners with different devices, contexts, and access needs.

Current commitments

Designed to remain usable without special access

The default site is static, public, and free to host. It should not require accounts, private data, paid services, or tracking scripts to read the atlas.

Semantic structure

Pages use landmarks, headings, lists, tables, links, skip navigation, and page metadata to support scanning and assistive technology.

Keyboard support

Navigation controls include visible focus states, mobile menu ARIA state, and Escape-key close behavior.

Non-map alternatives

The map is paired with county search, table links, downloadable CSV files, generated briefs, and fallback content.

Known constraints

Dense public data still needs careful presentation

County tables, map interactions, and metric-heavy brief pages can be difficult to interpret with assistive technology if copy, labels, or reading order drift. Accessibility improvements are welcome, especially for table semantics, map alternatives, keyboard flow, and plain-language summaries.

Feedback

Report an accessibility issue

Accessibility reports should include the page URL or generated file path, browser or device, assistive technology when relevant, expected behavior, and observed behavior.

Protect privacy

Do not include private, sensitive, or personally identifiable information in public issues. For sensitive data concerns, follow the security policy.
